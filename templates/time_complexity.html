Time Complexity — Made Simple

Understand algorithms like you understand daily routines.

Why do we need Time & Space Complexity?

Imagine a civil engineer checking a construction site. He measures cement ratio, sand ratio, and water ratio. Similarly, a programmer checks a code’s efficiency by measuring time and space.

Time Complexity tells how long a program takes to run as input grows.
Space Complexity tells how much memory it uses.

1️⃣ Big-Θ (Theta) Notation — Exact Growth

Think of Θ as the exact sandwich for runtime: it’s always between a lower and upper bound when input is large.

Example: Linear Search

var doLinearSearch = function(array, targetValue) {
  for (var guess = 0; guess < array.length; guess++) {
    if (array[guess] === targetValue) return guess;
  }
  return -1;
};

It runs at most n times (if value not found). Each iteration does constant work, so total = c1*n + c2. We ignore constants and low-order terms → Θ(n).

Key takeaway: Θ(f(n)) = tight bound. It shows growth rate precisely for large inputs.

2️⃣ Big-O (O) Notation — Upper Bound

Big-O tells us the maximum time an algorithm could take. It’s like saying: “It will take at most 1 hour.”

Example: Binary Search

function binarySearch(arr, target) {
  let start = 0, end = arr.length-1;
  while(start <= end) {
    let mid = Math.floor((start + end)/2);
    if(arr[mid] === target) return mid;
    else if(arr[mid] < target) start = mid+1;
    else end = mid-1;
  }
  return -1;
}

Worst-case runtime = O(log n). It’s an upper limit, not exact.

Tip: Big-O is useful to say: “Algorithm won’t be slower than this”

3️⃣ Big-Ω (Omega) Notation — Lower Bound

Big-Ω tells us the minimum time an algorithm will take. It’s like saying: “It will take at least 10 minutes.”

Example: Binary Search

Finding target on first guess → Ω(1).

Key takeaway: Big-Ω = best-case scenario. Big-O = worst-case. Θ = exact for large inputs.

Quick Summary

  • Θ(f(n)): Exact growth, tight bound.
  • O(f(n)): Upper bound, maximum time.
  • Ω(f(n)): Lower bound, minimum time.
